# SaaS開発インプット

SaaSを作る人として、プロダクト・UX・事業・運用・設計を深く見るための読み物を貯める場所です。

教養ノートの `daily/` とは分けて、こちらは自分の開発やプロダクト判断に直接つながるテーマを扱います。

## 使い方

SaaS向けノートを1本作る:

```sh
make saas
```

テーマを指定して作る:

```sh
make saas TOPIC="オンボーディングとは何か"
```

生成されるファイルは `001_テーマ.md`, `002_テーマ.md` のような連番形式です。

## 狙い

- SaaSを「機能の集合」ではなく「ユーザーの成果を支える仕組み」として見る
- 開発判断を、UX・事業・運用・データ・信頼の観点から考える
- 自分が作っているSaaSに引きつけて、明日見直せる問いを残す

## Codexに本文作成を頼むとき

```text
saas/001_テーマ.md を、SaaS開発者向けに読んで満足できる読み物として本文まで埋めて。
プロダクト設計、ユーザー理解、事業、運用、開発判断への接続を重視して。
```
