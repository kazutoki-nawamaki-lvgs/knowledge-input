# Product UI / UX Patterns

SaaSプロダクトで繰り返し現れる画面や操作パターンを、ユーザーの行動とプロダクト判断に結びつけて整理するノートを置く場所です。

扱うテーマは、一覧、検索、空状態、通知、オンボーディング、モーダル、エラー、設定、権限管理です。

トピック一覧は `topics/product_ui_ux_patterns_topics.csv` を参照します。

## 使い方

```sh
make product-ui-ux
```

テーマを指定する場合:

```sh
make product-ui-ux TOPIC="テーブルUIはなぜ設計が難しいのか"
```

## 書き方の方針

- `templates/practical.md` を起点にし、画面上の構造、ユーザーの行動、操作の詰まり方が見える本文にする
- 事業一般論や実装詳細に寄せすぎず、**ユーザーが画面で何を見て、何を判断し、どこで迷うか**を中心に扱う
- 具体例は、一覧、検索、フォーム、権限、通知など、SaaSで繰り返し現れる画面に引きつける
- 「実務への接続」では、改善できるUIの観察ポイント、情報設計、状態設計、操作導線に落とし込む
- 最後の問いは、実際の画面、問い合わせ、行動ログ、ユーザーテストで確かめられる粒度にする
