# Codex Instructions

このリポジトリで作業するときは、作業開始時に必ず次の2つを参照する。

- `README.md`: リポジトリ全体の構成、ノート生成方法、daily/ と saas/ の基本方針を確認する
- `saas/README.md`: SaaSノートの狙い、文章構成、品質基準、Codexへの依頼例を確認する

特に `daily/` や `saas/` のノートを作成・更新するときは、上記READMEの方針に加えて、該当する `topics/*.csv`、`templates/*.md`、既存の近い番号の記事を確認してから作業する。

## ノート作成時の基本方針

- 既存の番号・カテゴリ・ファイル名ルールを崩さない
- テンプレートを起点にしつつ、箇条書きだけで終わらせず、読み物として成立する本文まで書く
- SaaSノートでは、中心主張を絞り、プロダクト設計・ユーザー理解・事業・運用・開発判断への接続を明確にする
- dailyノートでは、問いから始まり、背景・具体例・現代への接続・持ち帰りが自然につながるようにする
