# Accessibility

SaaSプロダクトのアクセシビリティを、見た目の配慮ではなくプロダクト品質と運用の問題として整理するノートを置く場所です。

扱うテーマは、キーボード操作、色、フォーカス、スクリーンリーダー、フォーム、エラー表示、チーム運用です。

トピック一覧は `topics/accessibility_topics.csv` を参照します。

## 使い方

```sh
make accessibility
```

テーマを指定する場合:

```sh
make accessibility TOPIC="キーボード操作できるUIとは何か"
```

## 書き方の方針

- `templates/practical.md` を起点にし、利用可能性、支援技術、キーボード操作、フォーム、エラー、チーム運用のどこに効く話かを明確にする
- 善意や配慮の話だけで終わらせず、**ユーザーが実際に操作を完了できるプロダクト品質**として扱う
- 具体例は、フォーカス、ラベル、エラー表示、色だけに依存しない表現、スクリーンリーダー、モーダル、通知などを選ぶ
- 「実務への接続」では、実際の画面で確認できる操作、状態、読み上げ、レビュー観点に落とし込む
- 最後の問いは、キーボード操作、支援技術、フォーム送信、エラー復帰などで検証できる粒度にする
