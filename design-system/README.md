# Design System

SaaSプロダクトのUIを継続的に作り、直し、広げていくためのデザインシステムに関するノートを置く場所です。

扱うテーマは、コンポーネント、デザイントークン、Figmaとコードの接続、運用、変更管理、チーム内の合意形成です。

トピック一覧は `topics/design_system_topics.csv` を参照します。

## 使い方

```sh
make design-system
```

テーマを指定する場合:

```sh
make design-system TOPIC="デザイントークンは何を解決するのか"
```

## 書き方の方針

- `templates/practical.md` を起点にし、コンポーネント、トークン、設計原則、運用、変更管理のどこに効く話かを明確にする
- 個別画面の改善案だけで終わらせず、**チームが同じ品質でUIを作り続けるための仕組み**として扱う
- 具体例は、ボタン、フォーム、テーブル、色、余白、命名、Figmaとコードの同期など、再利用と変更に関わる題材を選ぶ
- 「実務への接続」では、デザインと実装のズレ、レビューの観点、破壊的変更、運用ルールに落とし込む
- 最後の問いは、自分たちのコンポーネント、トークン、ドキュメント、レビュー運用を確認できる粒度にする
